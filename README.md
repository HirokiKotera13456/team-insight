## 技術スタック

### 使用技術

- **Next.js 14** - React フレームワーク
- **Firebase Hosting** - 静的サイトホスティング
- **Firebase Authentication** - 認証（Gmail）
- **Cloud Firestore** - NoSQL データベース
- **Material-UI (MUI)** - UI コンポーネントライブラリ
- **Recharts** - チャートライブラリ
- **TypeScript** - 型安全性の確保
- **Jest** - テストフレームワーク
- **html2canvas / jsPDF** - PDF出力機能

### 技術選定理由

#### Next.js
Next.js は業界標準として広く利用されており、中長期運用にも耐えうる技術であるため選定しています。

- SSR/SSG によるパフォーマンス最適化が可能
- 豊富なエコシステムとコミュニティサポート
- 実務でも広く採用されている技術スタック

#### Firebase
個人開発規模でスピード感を持って開発するために必要だと感じたためです。

- 認証・データベース・ホスティングを統合的に提供
- フルスクラッチで認証システムを構築する必要がなく、開発効率が高い
- スケーラビリティとセキュリティが担保されている

## こだわった点

### 認証設計
認証には汎用的に利用されているGmail認証を用いました。また、短期開発を目的としているため、フルスクラッチで認証システムをつくらず、Firebase Authenticationを利用しました。

- **Gmail認証の採用**: ユーザーが既に持っているアカウントで簡単にログインできる
- **Firebase Authentication**: セキュアな認証を短期間で実装し、開発リソースを機能開発に集中できる

### UX設計
UXの観点では、なるべく利用者が煩雑な質問の回答を避けられるようにグラデーションで質問を回答できるようにしました。

- **スライダーによる直感的な回答方式**: 数値入力ではなく、視覚的に操作できるUIを実装
- **グラデーションUI**: 質問回答画面でグラデーションを使用し、視覚的に分かりやすいデザインを実現
- **エンジニアでなくても操作しやすい設計**: 専門知識がなくても直感的に理解・操作できるインターフェース

### 開発品質
- **TypeScript**: 型安全性を確保し、開発時のエラーを早期発見できる環境を構築
- **テスト**: Jestを使用したユニットテストで品質を担保
- **コンポーネント設計**: 再利用可能なコンポーネント設計により、保守性を向上

## プロジェクト概要

**自分自身の職場での立ち位置を理解するための可視化ツール**

Work Insight は、**仕事上の行動・判断の傾向を数値で可視化**することで、

- 自分自身の仕事の進め方の特徴を客観的に把握する
- 職場での自分の立ち位置や役割を理解する
- 自分の強みや傾向を数値で認識する

を、**エンジニアでなくても直感的に理解できる形**で示す自己理解のためのツールです。

> ⚠️ **注意事項**
> - 本ツールは性格診断ではありません
> - 「仕事の進め方の傾向」を整理し、自分自身の職場での立ち位置を理解するための補助ツールです
> - 良し悪しの評価は行いません

---

## 解決したい課題

> **「自分自身の職場での立ち位置を客観的に理解したい」**

仕事の現場では、

- 自分がどのような働き方をしているのか
- 職場で自分がどのような役割を担っているのか
- 自分の仕事の進め方の特徴は何か

といったことを、感覚ではなく数値で把握したいというニーズがあります。

本プロジェクトでは、これらを

👉 **感覚ではなく、数値と傾向** で自分自身を理解できる状態

を目指します。

## 主な機能

### 機能一覧

| 機能名 | 説明 | 実装内容 |
|--------|------|----------|
| **ログイン** | Gmail認証によるログイン | Firebase Authenticationを使用した認証機能 |
| **診断実施** | 12問の質問にスライダーで回答 | スライダーUIによる直感的な回答機能 |
| **スコア算出** | 4軸スコアを自動算出 | 回答値の正規化と加重平均によるスコア計算 |
| **結果表示** | レーダーチャート・棒グラフ・数値表示 | Rechartsを使用したデータ可視化 |
| **説明文生成** | 立ち位置・強みの文章自動生成 | スコアに基づいた動的な文章生成ロジック |
| **履歴保存** | 診断結果を自動保存 | Firestoreへのデータ永続化 |
| **履歴閲覧** | 過去の診断結果を時系列で確認 | 時系列チャートによる変化の可視化 |
| **PDF出力** | 診断結果をPDF形式で出力 | html2canvas + jsPDFによるPDF生成機能 |

### 軸スコアについて

各ユーザーは、以下の **4つの観点** について **0〜100 のスコア** を持ちます。

> **50 は「どちらでもない」状態です。**

| 観点 | 左寄り（低スコア） | 右寄り（高スコア） | 仕事で分かること |
|------|------------------|------------------|----------------|
| **行動エネルギー** | よく考えてから動く | まず動く | 決断の速さ |
| **判断基準** | 人の気持ち重視 | 論理・事実重視 | 意見のぶつかりやすさ |
| **進め方** | 柔軟に変えたい | 計画通り進めたい | 仕事の進行スタイル |
| **視点** | 現実・具体 | 抽象・将来 | 話の噛み合い方 |

👉 **良し悪しはありません**  
👉 「仕事の癖」を把握するための指標です

### 診断の流れ

1. **簡単な質問に答えるだけ**
   - 各観点につき3問（合計12問）
   - スライダー形式で直感的に回答

2. **スコアが算出される**
   - 入力結果は以下のような形で保存されます：

```json
{
  "行動エネルギー": 68,
  "判断基準": 42,
  "進め方": 81,
  "視点": 55
}
```

3. **結果を可視化**
   - **レーダーチャート**で自分の傾向を視覚的に確認
   - **立ち位置説明文**で自分の強み・役割・注意点を理解

4. **継続的な自己理解**
   - 診断履歴を保存し、時系列で変化を追跡
   - 職場環境や役割の変化が自分の傾向にどう影響するかを把握

## 開発の工夫

### 設計面での工夫

- **コンポーネント設計**: 再利用可能なコンポーネント設計により、保守性を向上
- **型安全性**: TypeScriptを徹底的に使用し、実行時エラーを削減
- **状態管理**: React Hooksを活用した効率的な状態管理

### パフォーマンス最適化

- **コード分割**: 動的インポートによるコード分割でバンドルサイズを削減
- **Firestoreの効率的なクエリ**: 必要最小限のデータ取得でパフォーマンスを向上

### セキュリティ

- **Firestore Rules**: ユーザーごとのデータアクセス制御を実装
- **認証必須**: 未認証ユーザーからのアクセスを防止
- **HTTPS通信**: すべての通信をHTTPSで保護

### テスト戦略

- **ユニットテスト**: ビジネスロジックのユニットテストを実装
- **コンポーネントテスト**: React Testing Libraryを使用したコンポーネントテスト
- **カバレッジ**: 重要なロジックのテストカバレッジを確保
